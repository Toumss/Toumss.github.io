Profil itinérant GPO

1) Dans la modification GPO on se rend dans “Ordinateur” (1), “Stratégie” (2), “Modèles d'administration” (3), “Système” (4), on commence par “Ouverture de session” (5) :

On va activer cette option :

L’ordinateur attendra la réponse réseau du serveur pour ouvrir la session utilisateur.

Le processus de synchronisation des dossiers prendra effet lors de l’ouverture de la session. Les dossiers itinérants seront créés lorsqu’un nouvel utilisateur se connectera sur le domaine.

3) Dans “Profil utilisateur” on s'attardera sur 3 paramètres :

Pour le premier paramètre "Ajouter le groupe de sécurité Administrateur aux profils utilisateur itinérants (1) :

 

 

Pour le second paramètre “Définir un chemin d'accès de profil itinérant pour tous les utilisateurs ouvrant une session”, activé en spécifiant un chemin réseau pour le dossier utilisateurs à sauvegarder :

 

La dernière stratégie spécifiera le chemin du dossier de base de l’utilisateur (ce qui créera un nouveau dossier pour chaque utilisateur. (Ici mon lecteur réseau spécifié est P et il est sur mon serveur) :

 

 

On pourra enfin dans les GPO par utilisateurs définir ce que l’on veut sauvegarder, ainsi que la limite de taille des dossiers :

 

  • Les dossiers seront créés lors de la première connexion de chaque nouvel utilisateur
  • Les administrateurs en auront le contrôle total
  • L’ouverture de session ne se fera pas si le serveur n’a pas répondu pour redescendre les fichiers des utilisateurs 

(On peut modifier encore quelques stratégies pour les connexions lentes / relancer N fois la demande au serveur…)
