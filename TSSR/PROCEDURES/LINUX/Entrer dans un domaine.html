Entrer dans un domaine

apt-get install -y adcli

 

Entrer dans un domaine :

Si le server à un server DHCP :
 

nano /etc/network/interfaces

Au dessus d'allow-hotplug ens33
auto ens33

Si le server AD n'a pas de DHCP : 

nano /etc/network/interfaces


Dans #The primary network interface
Ajouter : “auto ens33”


Pour retirer dhcp, enlever dhcp et mettre “static” a la place
Retour a la ligne + TAB “


address "192.168.240.15”
netmask “255.255.255.0”
gateway "192.168.240.2"
dns-nameservers “192.168.240.10” (Nom du server AD)
dns-nameservers “1.1.1.1”

une fois les modification effectuer “ctrl o” ; “crtl x”

systemctl restart networking

Installer les packet nécéssaire pour rentrer dans le domaine :

Commandes : 

apt install packagekit samba-common-bin sssd-tools sssd libnss-sss libpam-sss policykit-1 sssd ntpdate ntp realmd -y

Si le domaine possède un DHCP “oui” (1) sinon “non” (2) 

nano /etc/resolv.conf


Dans le dossier “Resolv.con” :
domain : nomdudomaine.com
nameserver “192.168.240.10” (Adresse server)

“Ctrl o” + “Ctrl x”

nano /etc/hostname


Dans le fichier hostname :
nomdelaVMLinux.nomdudomaine.com

“Ctrl o” + “Ctrl x”

realm join --user=administrateur “nomDuDomaine.com”


(Si ca fonctionne demande le mdp de l'admin du domaine)

 

Pour sortir du domaine utilisé la commande : 

realm leave "ad.example.com"
