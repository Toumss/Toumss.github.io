PRÉ-REQUIS

 

Toutes les commandes doivent être entrer en “root”

On commence “mettre à jour” les paquet sur la machine Debian 12 :

apt-get update && apt-get upgrade

Ensuite on s'attaque au “pré-requis” avant l'installation de GLPI, c'est a dire “Apache2”, php et mariadb-server :

apt-get install apache2 php mariadb-server -y

Puis, nous allons installer toutes les “extensions” nécessaires au bon fonctionnement de GLPI : 

apt-get install php-xml php-common php-json php-mysql php-mbstring php-curl php-gd php-intl php-zip php-bz2 php-imap php-apcu -y

Si on envisage d'associer GLPI avec un annuaire LDAP comme l'Active Directory, on doit installer l'extension LDAP de PHP : 

apt-get install php-ldap -y

Pour sécuriser un minimum mariadb on utilise la commande suivante :

mysql_secure_installation

On réponde “y” à tous et on suis les information qu'on nous donne.

Pour se connecter a MariaDB :

mysql -u root -p

On entre le mot de passe qu'on attribuer à l'étape précédente

On commence par crée la DataBase :

CREATE DATABASE DataBaseGLPI ;

On crée un utilisateur à qui on donne tous les privilèges :

GRANT ALL PRIVILEGES ON DataBaseGLPI.* TO glpi_admin@localhost IDENTIFIED BY "azerty";

On termine par la commande suivante, pour rafraichir les privileges :

FLUSH PRIVILEGES;

Pour sortir de MariaDb utiliser la commande :

EXIT
