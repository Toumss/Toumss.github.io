Hyperviseur

Qu'est-ce que c'est ?


Un hyperviseur, également connu sous le nom de superviseur de machine virtuelle, est un logiciel ou une couche de virtualisation qui permet à un système informatique physique (appelé hôte) d'exécuter plusieurs machines virtuelles (VM) en même temps.

L'hyperviseur isole les différentes machines virtuelles les unes des autres et les gère indépendamment, permettant ainsi à plusieurs systèmes d'exploitation et applications de fonctionner sur un seul matériel physique.

 

Il existe deux types principaux d'hyperviseurs

Hyperviseur de type 1 (ou bare-metal)

Ce type d'hyperviseur s'exécute directement sur le matériel physique de l'ordinateur sans nécessiter de système d'exploitation hôte supplémentaire.

Il contrôle l'accès aux ressources matérielles et gère les machines virtuelles directement. Des exemples d'hyperviseurs de type 1 incluent : 

  • VMware vSphere/ESXi
  • Microsoft Hyper-V 
  • KVM (Kernel-based Virtual Machine).

Hyperviseur de type 2 (ou hosted)

Ce type d'hyperviseur s'exécute sur un système d'exploitation hôte standard.

Il utilise les ressources du système d'exploitation hôte pour fournir des fonctions de virtualisation. 

Les machines virtuelles s'exécutent comme des processus sur le système d'exploitation hôte. 

Des exemples d'hyperviseurs de type 2 incluent : 

  • VMware Workstation 
  • VirtualBox 
  • Parallels Desktop.

 

 

 

Open Source Hypervisors

KVM (Kernel-based Virtual Machine)

KVM est une solution de virtualisation intégrée au noyau Linux, ce qui signifie qu'elle est directement intégrée dans le noyau Linux et est disponible gratuitement.

Elle est très populaire dans le domaine de la virtualisation d'entreprise et est souvent utilisée avec des outils de gestion tels que libvirt et oVirt.

Xen

Xen est un hyperviseur de type 1 open source qui peut être exécuté sur du matériel x86, x86-64 et ARM. 

Il est également intégré dans le noyau Linux, mais nécessite une couche de contrôle appelée Xen Project pour la gestion des VM.

Il est utilisé par de nombreux fournisseurs de cloud pour la virtualisation des serveurs.

VirtualBox

VirtualBox est un hyperviseur de type 2 open source développé par Oracle. 

Il peut être utilisé sur une variété de plates-formes hôtes, y compris Windows, macOS, Linux et Solaris. VirtualBox est populaire pour la virtualisation de bureau et le développement logiciel.

QEMU (Quick Emulator)

QEMU est un émulateur et un hyperviseur open source qui peut être utilisé pour la virtualisation. 

Il prend en charge une variété de processeurs hôtes et invités et peut être utilisé pour émuler des systèmes d'exploitation différents. 

KVM utilise également QEMU pour fournir la virtualisation matérielle.

 

Hyperviseurs Propriétaires

VMware vSphere/ESXi

VMware vSphere est une suite de virtualisation de serveur propriétaire qui comprend l'hyperviseur ESXi. 

Il est largement utilisé dans les environnements d'entreprise pour la virtualisation des serveurs et des ressources informatiques.

Microsoft Hyper-V 

Hyper-V est l'hyperviseur de Microsoft intégré dans les systèmes d'exploitation Windows Server. 

Il prend en charge la virtualisation de serveur et est largement utilisé dans les environnements Windows.

Citrix Hypervisor (anciennement XenServer) 

Citrix Hypervisor est une solution de virtualisation de serveur basée sur Xen, mais avec des fonctionnalités supplémentaires et un support commercial fourni par Citrix Systems.

Parallels Desktop for Mac 

Parallels Desktop est un hyperviseur de type 2 propriétaire conçu spécifiquement pour les utilisateurs de macOS. 

Il permet l'exécution de machines virtuelles Windows et d'autres systèmes d'exploitation sur un Mac.
